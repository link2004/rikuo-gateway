# リッチなコンテンツを提供する

## 概要

テキストだけでなく、ファイル参照、画像、URL、パイプ入力—あらゆる形式でコンテキストを提供できる。

---

## コンテンツ提供の方法

### `@`でファイルを参照

場所を説明する代わりに、ファイルを直接参照する。

```
@src/auth/handlers.ts
@README.md
```

Claudeは応答前にファイルを読み込む。

### 画像を直接貼り付け

- **コピー&ペースト**: スクリーンショットをそのまま貼り付け
- **ドラッグ&ドロップ**: 画像ファイルをプロンプトにドロップ

### URLを提供

ドキュメントのURLを与える。`/permissions`でドメインを許可リストに追加できる。

```
「このドキュメントを参考に実装して:
https://docs.example.com/api/v2」
```

### パイプでデータを渡す

```bash
cat error.log | claude
```

ファイル内容を直接送信できる。

### Claudeに自分で取得させる

Bashコマンド、MCPツール、ファイル読み取りを使って、Claudeに必要なコンテキストを自分で収集させることもできる。

```
「src/configディレクトリを読んで、
設定の構造を理解してから作業を開始して」
```

---

## まとめ

> 「リッチなコンテキストがリッチな結果を生む」

テキストだけでなく、ファイル参照、画像、URL、パイプ入力を組み合わせることで、Claudeに必要なすべてのコンテキストを提供できる。
